# Real OS

- All mharts goes to the same x0 address.
- at 0 copy everything that I want to execute in the OS.
	- OS image / kernel starts at 0.
		- It's part of bootloading by BIOS
- Everybody reads their own hartid.
- OkOk, I need to allocate my own PPDA.
- One hartid is master and all other are slaves.
	- The master(let it be hartid 0) allocates the PPDA for other hartids
		- Does by jumping to a determined address like 0x4000
		- And starts the PPDA from 0x5000
	- And then other hartids go to their PPDA and wait for the continuation message from the master.
		- So they start at 0x5000
		- one hartid takes it's PPDA size.
			- The other follow one by one.
		- So once the slave is released it goes to the PPDA.
- So before releasing the slaves,
	- It allocated the process table
		- say at 0x8000
	- each process will have a flag
		- Like is it picked up or not!
- Once the slaves are released
	- They are all going to the scheduler and it assigns the processes to the hartids.
